## Данные для дашборда

Приложение ожидает файлы в каталоге `dashboard/data/`.

| Файл | Назначение |
|------|------------|
| `wide_lsi.csv` или `wide_lsi.parquet` | Широкая таблица по дням: сырые и производные признаки M1–M5, колонки `LSI_*`, налоговые и прочие поля m4_* |
| `lsi_panel.csv` | Панель **глобального** LSI: как минимум колонки `date`, `lsi_raw` (предпочтительно) или `lsi_smooth`; сглаживание на дашборде согласовано с окном `SMOOTH_WINDOW` |
| `lsi_panel_local.csv` | Панель **локального** LSI (опционально; иначе ряд строится из колонки в `wide_lsi`) |
| `stress_events.csv` | События для вертикальных отметок на графиках: `date`, `label` |
| `module_drivers.json` | Необязательно: переопределение списка драйверов для RAG и карточек (слияние с встроенным fallback в коде) |
| `metrics.json` | Необязательно: метрики качества моделей для блока на главной |

Экспорт `wide_lsi` из ноутбука:

```python
import sys
from pathlib import Path

sys.path.insert(0, str(Path.cwd() / "dashboard"))
from export_from_notebook import export_wide_lsi

export_wide_lsi(wide_lsi)
```

Будет создан/обновлён `dashboard/data/wide_lsi.csv` и при наличии `pyarrow` — `wide_lsi.parquet`.

Если `wide_lsi` отсутствует, при запуске `utils.load_wide_lsi()` выбросит понятную ошибку с указанием пути.

## Страницы приложения

### Главная (`dashboard/app.py`)

- **Глобальный LSI** — модель в духе `LSI_lgbm_tuned`: уровень 0–100 относительно всей доступной истории в выгрузке.
- **Локальный LSI** — `LSI_lgbm_local_multi` (или отдельный CSV): «насколько сейчас необычно» на горизонте около года.
- Клик по точке на графике задаёт **дату разбора**; учитывается, с какого графика пришёл последний клик (глобальный/локальный).
- Блок драйверов: только «живые» сигналы (пороги по MAD и динамике), мини-графики, метрики, опционально японские свечи из дневного ряда.
- **Обзор дня (LLM)** — отдельный запрос в DeepSeek с JSON-ответом (общая картина, причины, события, горизонт).
- **Вклады модулей** — stacked area по колонкам `contrib_*` из панели, с эвристикой заполнения «дыр» по выходным.

### Модули (`dashboard/pages/01_📊_Модули.py`)

Вкладки M1–M5: ключевые ряды модуля, корреляции фич с выровненным по календарю LSI, опциональные свечи. Тексты от LLM не подменяют числа из данных.

### Аналитик (`dashboard/pages/02_🤖_Аналитик.py`)

Чат с **жёстким RAG**: в промпт попадают только заранее сгенерированные текстовые блоки из таблиц (период, агрегаты, топ-дни, драйверы, налоги, события, статические описания M1–M5). Парсер периода понимает русские формулировки, ISO и относительные запросы («последний год» и т.п.).

## LLM (DeepSeek)

Используется совместимый с OpenAI клиент (`openai` Python SDK) с `base_url=https://api.deepseek.com/v1`.

Задание ключа:

- через интерфейс **«Настройки»** на главной или в боковой панели на странице Аналитика;
- либо переменная окружения:

```bash
export DEEPSEEK_API_KEY="sk-..."
```

Модель по умолчанию в коде: `deepseek-chat`. Без ключа UI дашборда работает; блоки, которые вызывают LLM, показывают предупреждения или сырой контекст (на странице Аналитика).

## Ключевые константы и согласованность кода

| Константа | Где задаётся | Смысл |
|-----------|----------------|-------|
| `LSI_SHIFT` | `app.py` (+ отображение), должен совпадать с логикой в `utils.aligned_shifted_lsi_series_for_wide` через `LSI_DISPLAY_SHIFT` | Калибровочный сдвиг шкалы LSI для отображения (например, 20.0) |
| `SMOOTH_WINDOW` | `app.py`, `02_🤖_Аналитик.py`, `utils.LSI_SMOOTH_WINDOW` | Окно сглаживания (центрированное среднее), дней |
| `HIST_WINDOW` | `app.py` | Окно для медианы/перцентилей драйверов (~252 торговых дня) |
| `LSI_THRESHOLDS` | `utils.py` | Пороги зон 🟢 / 🟡 / 🔴 (по умолчанию 40 и 70) |

При изменении сдвига или окна сглаживания на главной странице **следует обновить те же значения** на странице Аналитика и в `utils`, иначе графики и RAG-контекст будут расходиться.

## Логика данных и предобработка

1. **Загрузка** `wide_lsi` из CSV или Parquet.
2. **Нормализация ставок m2**: если ключевая ставка в «старых» единицах (большие числа), выполняется пересчёт; пересчитываются спред и связанные MAD-поля.
3. **Forward-fill** непрерывных драйверов M1/M2/M3/M5 через выходные (исключая флаги и специальные m4/m5 поля), чтобы линии на графиках не имели разрывов там, где в казначейской логике значение «держится» до следующего обновления.
4. **Панель глобального LSI**: предпочтительно брать `lsi_raw` из `lsi_panel.csv` и сглаживать тем же `SMOOTH_WINDOW`, что и на графике; к отображаемому ряду добавляется `LSI_SHIFT`.

Подробные пользовательские формулировки метрик (медиана, «необычность», коридор на графике) задаются в `glossary.py` и подставляются в подсказки и LLM-лексикон.

## Модули M1–M5 (смысл в продукте)

| Модуль | Тема |
|--------|------|
| **M1** | Обязательные резервы, RUONIA |
| **M2** | Репо Банка России, cover ratio, спред ставки к ключевой |
| **M3** | Первичные размещения ОФЗ |
| **M4** | Налоговый календарь и сезонные эффекты |
| **M5** | Структурная ликвидность (ЦБ), потоки казначейства |

Точные имена колонок в данных — в ноутбуке и в `wide_lsi`; в интерфейсе аналитик не обязан их цитировать (см. системные промпты).

## RAG-чат «Аналитик»

1. Из текста запроса извлекается **период** (если есть) с ограничением по минимальной/максимальной дате в данных.
2. Определяется **интент**: обычный период, топ-стресса, «почему».
3. Собирается **контекст**: статические описания модулей + числовой блок из CSV (глобальный/локальный LSI, драйверы, события, тяжёлые налоговые дни).
4. В DeepSeek уходит system prompt с правилом **не выдумывать** цифры вне контекста.

История чата хранится в `st.session_state` до очистки или перезапуска сессии.

## Типовые проблемы

| Симптом | Что проверить |
|---------|----------------|
| `FileNotFoundError` для данных | Наличие `dashboard/data/wide_lsi.csv` или `.parquet` |
| Нет локального графика | Колонка `LSI_lgbm_local_multi` в `wide_lsi` или файл `lsi_panel_local.csv` |
| RAG пустой за период | Запрос выходит за пределы дат в `lsi_panel` / `wide_lsi` |
| LLM не отвечает | Ключ `DEEPSEEK_API_KEY`, сеть, баланс API |
| Расхождение LSI между главной и Аналитиком | Одинаковые `LSI_SHIFT`, `SMOOTH_WINDOW`, одна и та же колонка/панель |
